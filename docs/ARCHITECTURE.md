# Архитектура

## Обзор

```
                ┌──────────────────────────┐
   Telegram ───▶│  Telegraf (long polling) │── approve/reject платежей (bot.js)
                └──────────────────────────┘
                              │
 Пользователь                 ▼
   Mini App ──HTTPS──▶ ┌───────────────┐   ┌──────────────┐
   (public/)           │   Express     │──▶│ PostgreSQL   │
                       │  index.js     │   └──────────────┘
                       │  api.js       │──▶ GigaChat (ИИ)
                       └───────────────┘──▶ Yandex SpeechKit (голос)
```

Один процесс Node.js обслуживает и HTTP (Mini App + REST API), и Telegram-бота.

## Модули

| Файл | Ответственность |
|------|-----------------|
| `index.js` | Бутстрап: Express, пул PostgreSQL, инициализация схемы/миграций/индексов, бот Telegram, cron-задачи, graceful shutdown |
| `api.js` | REST API `/api/*`: статус, онбординг, генерация рецептов, история/избранное, режимы, платежи, админка, экспорт Excel |
| `auth.js` | Валидация подписи Telegram WebApp initData (HMAC-SHA256) |
| `gigachat.js` | OAuth-токен GigaChat с авто-обновлением и кэшем на диске; вызовы chat/completions с ретраями |
| `stt.js` | Распознавание речи через Yandex SpeechKit |
| `bot.js` | Inline-кнопки бота для подтверждения/отклонения платежей |
| `middleware/security.js` | Security-заголовки и in-memory rate-limiting |

## Данные (PostgreSQL)

- **users** — профиль, лимит бесплатных рецептов, аллергии, режим (standard/family/fitness), настройки уведомлений, бан.
- **subscriptions** — активные/прошлые подписки (PRO/VIP), срок действия.
- **payments** — заявки на оплату со статусом (`pending`/`approved`/`rejected`) и путём к чеку.
- **recipes** — сохранённые рецепты, избранное, рейтинг, счётчик приготовлений.
- **weekmenu_dishes** — история блюд из меню на неделю (чтобы не повторялись).

Схема, миграции и индексы создаются идемпотентно при старте (`index.js`).

## Поток оплаты

1. Пользователь загружает чек → `POST /api/payment/upload` создаёт запись `payments` (status `pending`).
2. Админу в Telegram приходит фото чека с кнопками «Одобрить»/«Отклонить» (`bot.js`)
   либо он действует через веб-админку (`/api/admin/payment/:id/...`).
3. При одобрении создаётся активная подписка на 30 дней, пользователю приходит уведомление.

## Фоновые задачи (cron, Europe/Moscow)

- `0 10 * * *` — уведомления об истекающих подписках + деактивация просроченных.
- `0 17 * * *` — ежедневное напоминание приготовить что-нибудь.
- `0 12 * * 3` — win-back пользователей, неактивных 5+ дней.
